### In a tweet

I built [eth-utils.com](https://eth-utils.com), a collection of common utilities for working with Ethereum codebases.

### Challenge

I often find myself googling online converters for [big numbers](https://eth-converter.com/), [checksum](https://ethsum.netlify.app/), [keccak256](https://emn178.github.io/online-tools/keccak_256.html) and other data types. I expected it was something most smart contract and JavaScript developers do when testing simple assertions or feature functionality.

### Solution

I interviewed a handful of React and Typescript developers working on Web3, and found that the most used tools were:

- Unit conversion (eg. wei, gwei)
- Checksum
- Hexstring conversion
- Keccak-256

### Results

I used the [geist-ui](https://geist-ui.dev/en-us) design system and built out these simple utils, you can see a demo below. The source code is available on [GitHub](https://github.com/albertocevallos/ethutils), free to create an issue if you'd like to see any new feature.
